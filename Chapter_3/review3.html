Terms

  1. Assignment
      Operation that assigns a value to a variable.
  2. cin
      "character input stream"
  3. Concatenation
      The joining of strings together like "snow" + "ball" is "snowball".
  4. Conversion
      Converting from one type like "int" to another type like "char".
  5. Declaration
      A statement that gives a name to an object(location in the memory).
  6. Decrement
      The operation of substraction.
  7. Definition
      A declaration that sets aside memory for an object.
  8. Increment
      The operation of addition.
  9. Initialization
      Giving an initial value to a variable.
  10. Name
      The name of a variable is preffered to be of letters, underscores, and numbers. It should not start with underscores or numbers, no double underscores in between the name and no spaces.
  11. Narrowing
      The conversion from a bigger type like "double" to a smaller type like "bool". The result is that some data is lost in that process.
  12. Object
      A location in the memory that can hold a value of a given type.
  13. Operation
      Any of the arithmetic operations. However, not all types have available all such operations.
  14. Operator
      The symbol that determines the type of operation. The operand is the value upon which the operation is performed.
  15. Type
      The type of value an object can hold. The object could only hold integers, single characters, string literals, and so on.
  16. Type Safety
      The practice of using objects only according to their rules that includes safely converting from one type to the other.
  17. Value
      A set of bits in memory interpreted according to a type. The data that an object holds.
  18. Variable
      A named object.

What is meant by the term prompt?

Encouraging the user to take action according to the instructions displayed from the program.

Which operator do you use to read into a variable?

The input operator ">>".

If you want the user to input an integer into your program for a variable named "number", what are two lines of code you could write to ask the user to do it and to input the value into your program?

What is "\n" called and what purpose does it serve?

It is a special character called "newline" and it moves the cursor to the next line.

What terminates input of a string?

"whitespace" which can be space, newline or tab characters.

What terminates input into an integer?

Characters that are not digits like letters and periods/commas.

cout << "Hello, " << first_name << "!\n";

What is an object?

It is a location in the memory that can hold certain type of values.

What is a literal?

Representation of values of certain types. You can have string literals like "Hello", integer literals like 2000, and so on.

What kind of literals are there?

String literals. integer literals, floating-point literals, single character literals, boolean literals, and pointer literals.

What is a variable?

A named object, that is, a location in the memory that contains a value and can be accessed through the use of that variable's name.

What are the typical sizes for a "char", an "int", and a "double"?

One byte, four bytes, and eight bytes.

What measures do we use for the size of small entities in memory, such as "int"s and "string"s?

The byte(8 bits).

What is the difference between "=" and "=="?

The first one assigns a value to a variable, the second one checks if two values are the same.

What is a definition?

A statement that sets aside memory for an object.

What is an initialization and how does it differ from an assignment?

Initialization assigns an initial value to a variable, assignment clears out the old value and gives it a new value.

What is string concatenation and how do you make it work in C++?

It is the joining of two strings, and it can be done with the use of the addition operator( + ).

Give examples of legal names that you shouldn't use because they are likely to cause confusion?

Names-names,foo-f00,fl-fI. Pretty much names that are short and use letters like i, o, because they can look like l, 0 respectively in some cases.

What are some good rules for choosing names?

Meaningful names that tell what they represent/do in our program as well as short.

What is "type-safety" and why is it important?

It is the practice of using objects according to their rules. You wouldn't want to substract "on" from the string literal "carbon" by use of the substraction operation. Nor would you want to convert double types into char because you are trying to fit a big chunk of memory into a small chunk resulting in loss of data.

Why can conversion from "double" to "int" be a bad thing?

Because the value of the double could be a value that requires a lot of memory, say 4.0000000000004, and when converted to an int, the exact value might have been lost since an int can't store that much memory like a double.

Define a rule to help decide if a conversion from one type to another is safe or unsafe?

If both types are of the same size then the conversion is safe, or if it's a conversion from a type of small memory to a type of a larger memory.
