Terms

  1. Argument error
      Functions, usually, take arguments. They expect those arguments to be of the correct data type the right number of them, and in the correct order too.
  2. Assertion
      A statement that states (asserts) an invariant(a condition that should always hold).
  3. catch
      When you throw an exception, you want to be able to catch it by writing a catch statement.
  4. compile-time error
      An error that happens during compilation. It is usually a syntax or type error.
  5. Container
      A collection of data, like the c++ vector.
  6. Debugging
      The process of finding errors, bugs, and removing them.
  7. Error
      A bug in the code.
  8. Exception
      It's an unprecedent event triggered while a program is run.
  9. Invariant
      A condition that should always hold.
  10. Link-time error
      An error that happens when several object files, translation units, can't be linked to create an executable program.
  11. Logic Error
      An error made by faulty reasoning, for example computing the age of someone and getting a negative number as an answer.
  12. post-condition
      A requirement of the return value of a function. We must make sure the output of a function is something with an expected criteria.
  13. pre-condition
      A requirement of a function upon its arguments.
  14. Range error
      If you exceed the range of a container, like a vector, then you will get this error.
  15. Requirement
      When we are dealing with functions, we must make sure the arguments meet some requirements. Otherwise, we can't use them, or if we use them but they produce an illogical answer then they fail the requirements. We can throw exceptions when the requirements are broken.
  16. run-time error
      An error that happens when the program is executed.
  17. Syntax Error
      An error that happens when we fail to use the specific programming language's rules, like forgetting to add a closing parentheses.
  18. Testing
      A systematic way of searching for errors using huge numbers of selected sets
  19. throw
      Reporting an unprecendet error.
  20. Type Error
      Using different data types than what they were expected to be.

What is a syntax error?

Am error due to mispelled statements that the compiler doesn't recognize as part of the programming language rules.

What is a type error?

What is a linker error?

If you call a function that has not been defined in any of your header files, then the linker displays this error because it can't find such function in your files.

What is a logic error?

What if your program finds the area of a figure by asking the user for some measurements but the user inputs a negative number, resulting in a negative area.

Compare and contrast having the caller of a function handle a run-time error vs. the called function's handling the run-time error.

If the caller handles the error, it results in messy code that tries to expose implemenation by the called function.

If the called function does it, it allows for checking to be done in one place as opposed to anywhere where the function was called, which would be the caller method. Checking the arguments in the function is better as opposed to the caller checking.

If a certain condition happens, the function throws an exception. It signals something went wrong. Then the catch part of the program will catch that exception, assumming the catch is expecting that specific exception, and then terminate the program or do something else.
