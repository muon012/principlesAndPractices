Terms

  1. //
      This token uses the "/" slash character and is used for writing comments in the source code.
  2. <<
      This is called the ouput operator.
  3. C++
      A programming language. That is, a precisely defined language with specific grammar and vocabulary for the kinds of actions we want performed by a computer.
  4. Comment
      Comments that are meant to describe what the program, or parts of it, does and other useful information for human readers of the code.
  5. Compiler
      The program that translate code (source code) written by a programmer into something (machine code) the computer can recognize.
  6. Compile-time error
      Error that occurs when the compiler is trying to compile the source code.
  7. cout
      "character output stream" is an standard output stream.
  8. Executable
      The machine code that is executed by the computer.
  9. Function
      A named sequence of instructions for the computer to execute in the order they were written.
  10. Header
      It is a file that contains definitions of terms that we use in our program.
  11. IDE
      "Integrated Development Environment." A programming environment with many options and features that allows you to write, excute, debug code easily.
  12. #include
      An include directive. It tells the computer to make available facilities from whichever file/library you wrote after "#include".
  13. Library
      Code that we access using declarions found in an #include file.
  14. Linker
      The program that links the different object codes to form an executable (machine code). This happens after the compiler compiled all the parts (translation units) of a program.
  15. main()
      This is where a C++ program starts its execution.
  16. Object code
      Machine code, code that has been already compiled (translated) from source code (the code the programmer writes).
  17. ouput
      Whatever it is displayed to the user.
  18. Program
      A set of instructions that tells the computer what to do.
  19. Source code
      The code you write, and see, as a programmer.
  20. Statement
      A part of a program that specifies an action.

Review Questions

What is the purpose of the "Hello, World!" program?

It is to get you acquainted with the tools of programming. This way we can see all the possible errors that might arise with improper syntax, IDE settings, grammar, and so on by using our computer and whichever programming environment we choose.

Name the four parts of a function?

Return type, name, parameters list, and function body.

Name a function that must appear in every C++ program?

main()

In the "Hello, World!" program, what is the purpose of the line "return 0"?

It is used to indicate that a program terminated succesfully.

What is the purpose of the compiler?

It translates source code to machine code. From human-readable to machine-readable.

What is the purpose of the "#include" directive?

It allows the program to use definitions from that file or library.

What does a ".h" suffix at the end of file name signify in C++?

That it is a header file. Headers contain definitions for the program.

What does the linker do for your program?

It links compiled files together in order to create an executable for a certain program. i.e. the "Hello_world" object code needs the "iostream library" object code linked by the linker before it creates the executable.

What is the difference between a source file and an object file?

Source file is what is written by the programmer, object code is what the machine can read and execute.

What is an IDE and what does it do for you?

"Integrated Development Environment" is a program that allows for code to be written edited, compiled, ran, debugged, and so on.

If you understand everything in the textbook, why is it necessary to practice?

Because repetition and practice are necessary for developing programming skills. Constant practice is the only way to develop and maintain high-level practical skills.
